# Environment Notes

## Project: instant-dev-environment-lab

This document explains the tools included in the development environment, why they are useful, and what skills the environment helps build.

The project is designed as a beginner-friendly portfolio lab for IT Support, Technical Support, and Junior Systems/Network Administration roles.

---

## 1. Environment Overview

The environment is designed to provide a consistent workspace that can run in:

- GitHub Codespaces
- VS Code Dev Containers
- A Docker-based local development setup

A consistent environment helps reduce setup problems because the tools and configuration are defined in the repository.

---

## 2. Installed Tools

### Git

Git is used for version control.

Why it is useful:

- Tracks file changes
- Supports collaboration
- Connects local work to GitHub
- Helps document project progress

Skills practiced:

- Checking status
- Committing changes
- Pushing to GitHub
- Reviewing project history

---

### curl

curl is used to test web requests from the terminal.

Why it is useful:

- Checks whether a website or API responds
- Tests HTTP headers
- Helps troubleshoot connectivity issues

Skills practiced:

- Basic web response testing
- API testing
- Connectivity troubleshooting

---

### wget

wget is used to download files from the command line.

Why it is useful:

- Downloads files directly in Linux
- Helps test file access
- Useful for setup and automation tasks

Skills practiced:

IT Support and Technical Support roles often require:

- Troubleshooting technical problems
- Documenting setup steps
- Working with command-line tools
- Understanding user environments
- Checking connectivity and configuration
- Communicating fixes clearly

This lab helps demonstrate those habits in a simple, honest, and portfolio-friendly way.

Junior Systems and Network Administration roles often involve:

- Working in Linux or server-like environments
- Reviewing logs and configuration files
- Testing connectivity
- Understanding automation basics
- Using Git for technical documentation
- Following repeatable setup procedures

This project introduces those concepts without overstating production experience.
